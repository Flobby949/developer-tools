## 1. 项目背景

### 1.1 历史背景
- 项目之前实现过去水印功能（手动工具：擦除、模糊、马赛克）
- 因技术限制被下线（commit a056ed2）
- imageUtils.ts 中保留了处理函数：`eraseArea`、`blurArea`、`mosaicArea`
- 当前 WatermarkRemover.vue 仅显示"功能不可用"提示

### 1.2 新需求
- **功能类型**: 纯 AI 智能去水印
- **部署策略**: 渐进式方案（先前端算法，后续可升级到真正的 AI 服务）
- **用户体验**: 预览确认式（AI 标记 → 用户确认 → 处理）
- **处理方式**: 仅使用擦除功能

### 2.2 核心流程

1. **上传阶段**: 用户上传图片 → 显示原图
2. **检测阶段**: 点击"检测水印" → 调用检测服务 → 在图片上标记检测区域
3. **确认阶段**: 用户可以调整/删除/添加标记区域
4. **处理阶段**: 点击"处理水印" → 使用擦除功能处理选中区域
5. **下载阶段**: 下载处理后的图片

## 4. 前端检测算法设计

### 4.1 检测策略（多层检测）

**策略1: 颜色异常检测**
- 原理: 水印通常颜色单一且与背景对比明显
- 方法:
  - 将图片分成网格（如 32x32 像素块）
  - 计算每个块的颜色方差
  - 低方差 + 高对比度 = 可能是水印

**策略2: 边缘密度检测**
- 原理: 文字水印边缘密集
- 方法:
  - 使用 Sobel 算子检测边缘
  - 计算局部区域的边缘密度
  - 高密度区域 = 可能是文字水印

**策略3: 位置模式检测**
- 原理: 水印通常在固定位置（四角、中心）
- 方法:
  - 优先检测图片四角和中心区域
  - 对这些区域提高检测权重

### 4.2 检测参数

```typescript
interface DetectionConfig {
  sensitivity: number      // 0-1, 灵敏度
  minRegionSize: number   // 最小区域大小（像素）
  maxRegionSize: number   // 最大区域大小（像素）
  confidenceThreshold: number  // 置信度阈值
}

// 默认配置
const DEFAULT_CONFIG = {
  sensitivity: 0.5,
  minRegionSize: 20,
  maxRegionSize: 500,
  confidenceThreshold: 0.6
}
```

### 4.3 算法输出

- 返回检测到的区域数组
- 每个区域包含: 位置、大小、置信度
- 按置信度降序排列
- 过滤掉置信度低于阈值的区域

### 4.4 性能优化

- 使用 Web Worker 处理图像分析（避免阻塞 UI）
- 对大图片先缩小处理，再映射回原图坐标
- 缓存中间计算结果

## 6. 错误处理和边界情况

### 6.1 错误处理

**文件上传错误**:
- 文件格式不支持 → Toast 提示"仅支持 PNG、JPG、WebP 格式"
- 文件过大（>20MB） → Toast 提示"文件大小不能超过 20MB"
- 文件损坏 → Toast 提示"图片加载失败，请检查文件"

**检测错误**:
- 未检测到水印 → Toast 提示"未检测到明显水印，可手动添加区域"
- 检测超时 → Toast 提示"检测超时，请重试"
- 内存不足 → Toast 提示"图片过大，建议先压缩"

**处理错误**:
- 未选择区域 → Toast 提示"请至少选择一个区域"
- 处理失败 → Toast 提示"处理失败，请重试"

### 6.2 边界情况

**极端图片尺寸**:
- 超大图片（>4000px） → 先缩放到合理尺寸检测，处理时用原图
- 超小图片（<100px） → 提示"图片太小，检测效果可能不佳"

**特殊水印类型**:
- 透明水印 → 检测困难，提示用户手动添加
- 平铺水印 → 可能检测出多个区域，需要用户确认
- 复杂背景 → 可能误判，依赖用户调整

**性能优化**:
- 大图片检测时显示进度条
- 使用 requestIdleCallback 分批处理
- 限制同时处理的区域数量（<50个）

### 6.3 用户引导

**首次使用**:
- 显示简单的使用提示
- 示例: "1. 上传图片 → 2. 点击检测 → 3. 确认区域 → 4. 处理下载"

**检测结果为空**:
- 提示: "未检测到水印？可以手动拖拽添加区域"

**置信度说明**:
- 在区域列表旁显示: "百分比表示检测置信度，建议保留 80% 以上的区域"

## 9. 风险和限制

### 9.1 技术风险

- 前端算法检测准确率有限，可能需要用户大量手动调整
- 复杂水印（透明、渐变、嵌入式）检测困难
- 大图片处理可能导致浏览器性能问题

### 9.2 用户体验风险

- 如果检测效果不佳，用户可能失去信心
- 手动调整区域的操作可能不够直观

### 9.3 缓解措施

- 设置合理的用户期望（提示这是辅助工具）
- 提供清晰的操作指引
- 保留手动添加区域的能力
- 后续可升级到真正的 AI 服务

## 10. 成功指标

- 检测准确率 > 60%（简单水印）
- 用户完成整个流程的成功率 > 80%
- 处理时间 < 5秒（中等尺寸图片）
- 用户满意度反馈
